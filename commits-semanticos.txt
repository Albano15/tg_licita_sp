feat: nova funcionalidade -> ex: git commit -m "feat(auth): adiciona login com Google"

fix: correção de bug ->  ex: git commit -m "fix(cart): corrige cálculo do total ao remover item"

docs: documentação -> ex: git commit -m "docs(readme): atualiza instruções de instalação"

style: formatação, semântica, espaços, etc -> ex: git commit -m "style(button): ajusta padding e cores de hover"

refactor: refatoração de código sem alterar comportamento -> ex: git commit -m "refactor(user-service): separa funções de validação em módulo próprio"

test: adição ou correção de testes -> ex: git commit -m "test(api): adiciona testes para endpoint de usuários"

chore: tarefas de manutenção, build, config, etc. -> git commit -m "chore(deps): atualiza dependências do projeto"
